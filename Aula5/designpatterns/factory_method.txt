- A ideia do factory method é centralizar a criação de objetos.
Por exemplo:

class UserService {
    public function __construct($a, $b, $c) {

    }
}

$userService = new UserService($a, $b, $c);

- Caso eu precise adicionar mais uma dependência na classe UserService, eu precisarei alterar o código em todos os lugares onde a classe é instanciada.

No uso do factory method, eu criaria uma classe UserServiceFactory, que seria responsável por criar a instância de UserService:

class UserServiceFactory {
    public static function make() {
        return new UserService(new A(), new B(), new C());
    }
}

$userService = UserServiceFactory::make();

- Ja o Singleton (Anti-pattern), sempre que for chamado vai criar uma nova instância da classe, e caso a instância já exista, ele retorna a mesma instância.

class UserServiceSingleton {
    protected static object $instance;

    public static function make() {
        static::instance = static::instance ?? new UserService(new A(), new B(), new C());

        return static::instance;
    }
}
